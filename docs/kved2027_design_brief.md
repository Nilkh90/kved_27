## 1. О проекте

**kved2027** — веб-сервис для перехода украинского бизнеса с классификатора КВЕД-2010 на новый стандарт NACE 2.1-UA, который вступает в силу в 2027 году (Наказ Держстату № 191).

Сервис решает конкретную проблему: бухгалтер, ФОП или руководитель вводит свой старый код КВЕД — и за 10 секунд получает ответ на вопрос «что изменится и нужно ли идти к регистратору».

**Ключевой UX-принцип**: интерфейс работает с бюрократической информацией, которая вызывает тревогу. Задача дизайна — снижать стресс, а не добавлять его. Ясность и спокойствие важнее «wow-эффекта».

## 4. Технический контекст (важно для дизайнера)

- **Стек**: PHP / Laravel + Livewire + Blade + Tailwind CSS
- **Рендеринг**: серверный (SSR). Нет React, нет SPA-переходов — обычная навигация по страницам
- **Анимации**: реализуются через Alpine.js + CSS transitions. Возможны простые микроанимации (раскрытие дерева, появление dropdown). Сложные JS-анимации — нежелательны
- **Язык интерфейса**: только украинский
- **Шрифты**: предпочтительны Google Fonts или системные (без сторонних платных лицензий)
- **Иконки**: любой открытый набор (Heroicons, Lucide, Phosphor) — согласовать с разработчиком

## 5. Визуальное направление

### Стиль
Clean Corporate UI — современный, строгий, дружелюбный. Ориентир — лучшие европейские государственные цифровые сервисы (gov.uk Design System, Дія), но без казённости. Пространство, белый фон, чёткая иерархия.

**Не нужно**: агрессивные градиенты, тёмная тема (не в скоупе), иллюстрации в стиле «стартап-сайт», перегруженные анимации.

### Обязательная цветовая система

Дизайнер может предложить оттенки и акценты, но логика цветов — фиксирована, так как она отражает юридическую суть данных:

| Роль | Смысл | Направление |
|---|---|---|
| **Primary** | Основной бренд, CTA, активные элементы | Оттенки синего (доверие, госсервис) |
| **Success (зелёный)** | Код переходит автоматически, ничего делать не нужно | Зелёный |
| **Warning (жёлтый/оранжевый)** | Код разделился, нужно выбрать новое направление | Янтарный / оранжевый |
| **Danger (красный)** | Требуется подать заявление регистратору | Красный |
| **Neutral** | Фоны, границы, вспомогательный текст | Серый |

Эти четыре статуса — основа UX. Пользователь должен понимать свой статус одним взглядом, без чтения текста.

## 6. Компоненты: детальное описание

### 6.1. StatusBadge — ключевой элемент проекта
Используется везде, где отображается статус перехода кода. Три варианта + один критический:

```
🟢 «Автоматичний перехід»     — зелёный  (transition_type: 1_TO_1)
🟡 «Потрібен вибір напряму»   — жёлтый   (transition_type: 1_TO_N)  
🔵 «Коди об'єднано»           — синий    (transition_type: N_TO_1)
🔴 «Потрібна перереєстрація»  — красный  (action_required: true, перекрывает всё)
```

Размеры: стандартный (в карточках и таблицах) и крупный (заголовок детальной страницы).

### 6.2. SearchBar — главный инструмент сервиса
Появляется в двух контекстах:
- **Hero-версия** на главной: крупная, занимает центр экрана, с подсказкой-плейсхолдером
- **Компактная** в шапке на внутренних страницах

Поведение: при вводе появляется dropdown с подсказками (до 8 результатов). Каждая подсказка содержит код (моноширный шрифт), название и статус-бейдж. Отдельная кнопка «Описати бізнес» — для AI-поиска (фаза 2, но место в интерфейсе зарезервировать).

### 6.3. MappingPanel — блок сравнения «до / після»
Ключевой блок на детальной странице кода. Визуально разделён на две части:

```
┌─────────────────────┬─────────────────────┐
│   КВЕД 2010         │   NACE 2027         │
│   62.01             │   J62.010           │
│   Розробка ПЗ       │   Розробка ПЗ       │
│   [статус-бейдж]    │   [статус-бейдж]    │
└─────────────────────┴─────────────────────┘
```

При типе 1_TO_N (один код разделился на несколько) — правая часть содержит несколько вариантов с чекбоксами для выбора.

### 6.4. ClassifierTree — иерархическое дерево
Используется на странице каталога. Структура: Секція → Розділ → Група → Клас.

Раскрытие анимированное (CSS transition). Активный узел подсвечен. Рядом с каждым узлом — маленький статус-индикатор (есть ли изменения в этом разделе). Переключатель вверху: «КВЕД 2010» / «NACE 2027».

### 6.5. CodeCard — карточка в результатах поиска
Компактная карточка с: кодом (моноширный), названием, статус-бейджем и стрелкой перехода.

### 6.6. PopularChanges — таблица на главной
Интерактивная таблица: «Старий код → Новий код → Статус». 10 строк, каждая строка кликабельна и ведёт на детальную страницу. Показывает суть сервиса без единого слова объяснений.

## 7. Страницы: поэкранное описание

### 7.1. Главная страница
**Цель**: за 5 секунд донести ценность и направить в поиск.

Секции сверху вниз:
1. **Шапка**: логотип kved2027 слева, навигация справа (Каталог, Методологія), кнопка CTA
2. **Hero-блок**: крупный заголовок (суть проблемы в 1-2 строках), подзаголовок (суть решения), крупная строка поиска. Без декоративных иллюстраций — чистота
3. **Блок ценности**: 4 карточки-тезиса. Тезисы: (1) штрафы за несвоевременную смену, (2) дедлайн 2027, (3) для 80% кодов ничего делать не нужно, (4) проверка бесплатна. Иконки минималистичные
4. **Таблица популярных изменений**: заголовок «Найпопулярніші зміни», таблица из 10 строк (код → код → статус), ссылка «Переглянути всі»
5. **CTA-блок**: две кнопки — «Перейти до каталогу» (primary) и «Читати методологію» (outline)
6. **Футер**: логотип, навигация, копирайт, ссылка на Наказ Держстату № 191

### 7.2. Страница результатов поиска
**URL**: `/search?q=62.01`

Макет: двухколоночный на десктопе (фильтры слева, результаты справа), одноколоночный на мобильном.

Левая панель (фильтры):
- Фильтр по стандарту: КВЕД 2010 / NACE 2027 / Всі
- Фильтр по статусу: Автоматичний / Потрібен вибір / Потрібна дія

Правая часть:
- Строка поиска (компактная версия) вверху с текущим запросом
- Количество результатов: «Знайдено 12 результатів для "62.01"»
- Список CodeCard-карточек
- Состояния: загрузка (скелетон), пустой результат («Нічого не знайдено — спробуйте інший запит»)

### 7.3. Детальная страница кода ← самая важная
**URL**: `/code/kved/62.01`

Структура:
1. **Хлебные крошки**: Головна → Каталог → Секція J → 62.01
2. **Шапка кода**: 
   - Крупный код моноширным шрифтом (напр. `62.01`)
   - Название кода
   - Крупный StatusBadge
   - Кнопки действий: «Зберегти», «Роздрукувати», «Поділитись»
3. **MappingPanel**: блок «Як було → Як стало» (описан в 6.3)
4. **Блок описания**: вкладки «Включає» / «Не включає» с текстом и гиперссылками на смежные коды
5. **Комментарий к переходу**: текстовый блок с объяснением почему произошло изменение (из поля `transition_comment`)
6. **Блок «Що робити»**: конкретные шаги для пользователя в зависимости от статуса. При `action_required=true` — выделенный красный блок с инструкцией
7. **Смежные коды**: горизонтальный скролл карточек

### 7.4. Каталог классификатора
**URL**: `/catalog`

Переключатель вверху: КВЕД 2010 | NACE 2027

Основная область: дерево ClassifierTree (описано в 6.4). На десктопе — двухколоночный макет: дерево слева, превью выбранного узла справа. На мобильном — одна колонка, при клике переход на детальную страницу.

### 7.5. Методология / FAQ
**URL**: `/info`

Аккордеон с вопросами-ответами. Группировка по темам: «Загальне», «Строки та штрафи», «Процедура перереєстрації», «Технічні питання». Сайдбар со ссылками на официальные документы (Наказ № 191 и др.).

## 9. Микроанимации

Технически реализуются через Alpine.js + CSS transitions. Дизайнер описывает поведение, разработчик реализует.

Обязательные:
- Раскрытие/закрытие узлов дерева: `max-height` transition, 200ms ease
- Появление dropdown поиска: fade + slide down, 150ms
- Hover-состояния всех кликабельных элементов
- StatusBadge: без анимации, статичный (важность информации не должна «мигать»)

Нежелательные: анимации при загрузке страницы (мешают SEO-восприятию и раздражают при повторных визитах).

## 10. Типографика

Требования:
- Читаемость украинских текстов с кириллицей
- Обязателен **моноширный шрифт** для отображения кодов (`62.01`, `J62.010`) — должен быть чётко отличим от обычного текста
- Google Fonts или системные (без платных лицензий)
- Поддержка: украинский кириллический набор символов

Шкала (минимальная, дизайнер может расширить):

| Роль | Применение |
|---|---|
| Display / Hero | Главный заголовок на главной |
| H1 | Заголовок страницы |
| H2 | Заголовок секции |
| H3 | Заголовок карточки |
| Body | Основной текст описаний |
| Small | Метаданные, подписи |
| Code | Коды классификатора (моноширный) |
